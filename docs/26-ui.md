# UI

`@rovy/ui` is the Rovy UI package. It is a TypeScript-authored, Rovy-owned immediate UI runtime with an EgooE/Plasma-style authoring feel.

This design is intentionally:

- full Roblox TS authoring
- function-first
- callsite-scoped widget storage for helpers like `useState`, `useEffect`, and `useInstance`
- not class-based

It is **not** a React clone, **not** a public Lua-authored surface, and **not** a dependency or re-export of `@rbxts/egooe`. EgooE is behavior reference only.

## Package shape

`@rovy/ui` is a separate runtime package, parallel to `@rovy/networking`.

- `@rovy/core` stays the ECS/runtime package
- `@rovy/networking` stays the net-event package
- `@rovy/ui` is the widget/render integration package
- `rovy-transformer` handles the compile-time widget lowering

The public authoring story stays entirely TypeScript-authored even though roblox-ts eventually emits Luau.

## Frame lifecycle

Every render pass requires a root node tied to a Roblox `Instance`. Call `start` each frame to re-render.

```ts
import RovyUi from "@rovy/ui";

const root = RovyUi.new(screenGuiInstance);

// re-run every frame / on demand
RovyUi.start(root, () => {
	RovyUi.window("My Window", () => {
		RovyUi.label("Hello world");
	});
});
```

For split begin/end (e.g. yielding work between):

```ts
const handle = RovyUi.beginFrame(root, myRenderFn);
// ... do other work ...
RovyUi.continueFrame(handle, moreRenderFn);
RovyUi.finishFrame(root); // prunes stale nodes
```

### API

| Function | Signature | Notes |
|---|---|---|
| `RovyUi.new` | `(rootInstance: Instance) => Node` | creates root node |
| `start` | `(node, callback, ...args) => void` | begin + finish in one call |
| `beginFrame` | `(node, callback, ...args) => ContinueHandle` | open frame, no prune yet |
| `continueFrame` | `(handle, callback, ...args) => void` | additional work in same frame |
| `finishFrame` | `(node) => void` | prune stale child nodes |

## Storage helpers

All storage helpers must be called inside a widget body (inside a running frame). State is scoped to the callsite — two separate calls to the same widget function at different call sites have independent state.

### `useState`

```ts
useState<T>(initialValue: T | (() => T)): [T, StateSetter<T>]
```

Persists a value across frames for the current callsite. The setter accepts either a new value or an updater function.

```ts
/** @widget */
export function Counter(): void {
	const [count, setCount] = useState(0);
	label(`Count: ${count}`);
	if (button("Increment").clicked()) setCount((n) => n + 1);
}
```

The value survives re-renders; it is destroyed when the widget node is pruned (i.e. the call is no longer reached).

---

### `useEffect`

```ts
useEffect(callback: () => void | (() => void), ...dependencies: defined[]): void
```

Runs `callback` when dependencies change. If `callback` returns a function, it is called as cleanup before the next run or when the node is destroyed.

```ts
/** @widget */
export function AudioPlayer(trackId: string): void {
	useEffect(() => {
		const sound = playTrack(trackId);
		return () => sound.Stop();
	}, trackId);
}
```

Omitting dependencies runs the effect every frame.

---

### `useInstance`

```ts
useInstance<T extends object>(
	creator: (ref: Record<string, Instance>) => Instance | [Instance | undefined, Instance?]
): T
```

Creates a Roblox `Instance` once; reuses it on subsequent frames; destroys it when the node is pruned. The `ref` object passed to `creator` can be used to collect named handles.

Return a plain `Instance` to set it as the node's render target. Return `[instance, container]` to separate the mounted instance from the child-parenting container.

```ts
/** @widget */
export function Badge(text: string): void {
	const refs = useInstance((ref) =>
		create("TextLabel", {
			[ref as never]: "label",
			Size: new UDim2(0, 80, 0, 24),
		}),
	) as { label: TextLabel };

	refs.label.Text = text;
}
```

## `useKey` — per-item discriminators

`useKey` refines the identity of the current frame before widget calls. When the same widget is called multiple times at the same source location (a loop), each iteration needs a distinct stable key so their internal state stays separate.

```ts
useKey(discriminator: string | number): void
```

Call `useKey` once per loop iteration, before the widget that needs per-item identity:

```ts
for (const item of inventory) {
	useKey(item.id);
	const cb = checkbox(item.name, { checked: item.equipped });
	if (cb.clicked()) toggleEquip(item.id);
}
```

`useKey` is sticky for the rest of the current frame at that level. In a loop this is fine because each iteration replaces it. Outside a loop, prefer `StyleScope` with a `discriminator` field when you only need scoped style, as it is self-contained.

## Context API

Contexts pass values down the call tree without threading them through every widget signature.

```ts
createContext<T>(name: string): Context<T>
provideContext<T>(context: Context<T>, value: T): void
useContext<T>(context: Context<T>): T | undefined
```

```ts
const ThemeCtx = createContext<"light" | "dark">("Theme");

/** @widget */
export function ThemeProvider(mode: "light" | "dark", children: () => void): void {
	provideContext(ThemeCtx, mode);
	children();
}

/** @widget */
export function ThemedLabel(text: string): void {
	const mode = useContext(ThemeCtx) ?? "dark";
	label(text, { color: mode === "dark" ? Color3.fromRGB(255,255,255) : Color3.fromRGB(0,0,0) });
}
```

`useContext` walks up the frame stack and returns the nearest provided value.

## Style

### Active style

Every widget implicitly reads the current active style. Widgets authored with a leading `style: Style` parameter have that parameter erased by the transformer; the body instead reads `RovyUi.getActiveStyle()` automatically.

```ts
/** @widget */
export function MyPanel(style: Style, title: string): void {
	// style is injected at compile time — callers write MyPanel("Title")
}
```

```ts
useStyle(): Style       // read active style inside widget body
getActiveStyle(): Style // alias
```

The `Style` interface covers all visual tokens: colors, transparencies, sizes, paddings. See [`src/style.ts`](../packages/ui/src/style.ts) for the full shape.

---

### `setStyle`

Merges a partial patch onto the current active style for the rest of the current frame scope.

```ts
setStyle(patch: Partial<Style>): void
```

Prefer `StyleScope` over a raw `setStyle` call — `setStyle` is unbounded and affects all subsequent widget calls in the frame.

---

### `StyleScope`

```ts
StyleScope(options: StyleScopeOptions, fn: () => void): void
withStyleScope(options: StyleScopeOptions, fn: () => void): void // identical
```

```ts
interface StyleScopeOptions {
	patch: Partial<Style>;
	discriminator?: string | number;
}
```

Merges `patch` onto the current style for exactly the duration of `fn`. When `fn` returns, the parent style is restored automatically.

```ts
StyleScope({ patch: { textColor: Color3.fromRGB(255, 80, 80) } }, () => {
	label("Danger zone");
	button("Delete");
});
```

Use `discriminator` when the same `StyleScope` call site is reached multiple times with different data (e.g. inside a list loop):

```ts
for (const item of items) {
	StyleScope(
		{
			patch: { textColor: item.isRare ? Color3.fromRGB(255, 220, 120) : defaultStyle.textColor },
			discriminator: item.id,
		},
		() => {
			label(item.name);
		},
	);
}
```

## `create` — instance helper

```ts
create<T extends keyof CreatableInstances>(
	className: T,
	props?: Record<string | number, unknown>
): CreatableInstances[T]
```

Thin wrapper over `new Instance(className)` that applies properties and supports a few conventions:

| Key type | Value type | Behavior |
|---|---|---|
| `string` | any | sets property by name |
| `string` | `function` | connects to signal or fires event callback |
| `number` | `Instance` | parents instance at that index |
| `object` (ref) | `string` | stores instance into ref under that name |

```ts
useInstance((ref) =>
	create("Frame", {
		[ref as never]: "container",
		BackgroundTransparency: 1,
		Size: new UDim2(1, 0, 1, 0),
		0: create("UIListLayout", { SortOrder: Enum.SortOrder.LayoutOrder }),
	}),
) as { container: Frame };
```

`AutoLocalize` is automatically set to `false` for all 2D GUI classes.

## Custom widgets

Tag a function with `/** @widget */` and the transformer handles registration and call lowering.

```ts
/** @widget */
export function HealthBar(style: Style, current: number, max: number): void {
	progressBar({ value: current / max, label: "HP" });
	label(`${current} / ${max}`, { color: style.textDisabledColor });
}
```

Call it like any built-in:

```ts
HealthBar(player.health, player.maxHealth);
```

Rules:

- `/** @widget */` must be on a `function` declaration, not an arrow or variable.
- A same-file implementation is required. Overloads are fine; the transformer picks the one with a body.
- `style: Style` as the first parameter is optional authoring sugar — removed from the call signature at compile time.
- `useState`, `useEffect`, `useInstance`, `useKey`, `createContext`/`useContext`/`provideContext`, and `StyleScope` all work inside widget bodies.
- Widgets must not yield.

```ts
/** @widget */
export function ItemRow(style: Style, item: Item): void {
	const [selected, setSelected] = useState(false);

	StyleScope({ patch: { frameBgColor: selected ? style.headerColor : style.frameBgColor } }, () => {
		row(() => {
			if (button(item.name).clicked()) setSelected((v) => !v);
			label(`x${item.count}`, { color: style.textDisabledColor });
		});
	});
}
```

## Authoring model

```ts
/** @widget */
export function Window(props: WindowProps): void;
export function Window(style: Style, props?: WindowProps): void {
	if (!props) return;
	print(style.windowBgColor, props.title);
}

Window({ title: "Inventory" });
```

For a clean public call signature, use overloads: the `/** @widget */` tag goes on the overload without a body; the implementation overload carries `style: Style` and the body.

There is no widget-class contract. Do not write `new Window()` or `RovyUi.Window(...)` from consumer code.

---

## Transformer contract

The transformer owns widget discovery and call lowering.

- Detects `/** @widget */` on function declarations and `const x = widget(...)` declarations.
- Requires a same-file implementation for tagged functions authored in consumer code.
- Injects widget registration metadata as a module side effect (consumer-authored widgets only).
- Wraps consumer-authored widget functions through `RovyUi.__widget(fn, meta)`.
- Lowers widget calls `Widget(args)` to `RovyUi.__scope("module:key", () => Widget(args))` — stable callsite identity with no array allocation.
- Storage-bearing calls lower to keyed internals: `useState(x)` → `RovyUi.__useState("module:key", x)`.
- `scope(fn)` lowers to `RovyUi.__scope("module:key", fn)`.
- Leading `style: Style` param is removed from the runtime signature; body is prefixed with `const style = RovyUi.getActiveStyle()`.

### Two keys

Mirroring EgooE's `Runtime.widget` / `scope` split (EgooE derives both from `debug.info`; Rovy bans `debug.info` and injects both at compile time):

- **Widget identity key** — identifies the widget *definition*. Carried in the registry meta `id`: `RovyUi.__widget(fn, { id, name })`. For consumer widgets the transformer computes it from the declaration; for built-ins it is the literal passed to `widget(fn, "@rovy/ui/button")` inside the library.
- **Callsite key** — identifies each *call* to a widget. `RovyUi.__scope("module:N", () => widget(args))`, where `N` increments per widget callsite in source order. This is what gives loops and repeated calls stable, separated storage. `useKey(discriminator)` further refines it at runtime for per-item identity.

### Built-in widget detection

Built-in `@rovy/ui` widgets are detected the **same way** as consumer widgets: via the `/** @widget */` JSDoc tag. The tag is preserved into the package's emitted `.d.ts` (on `export declare const button`, on the `_table` alias target, on `function` overloads). The consumer-side transformer resolves the `@rovy/ui` export through the type checker, follows re-export/alias chains, and lowers a call to `RovyUi.__scope(...)` when the resolved declaration carries `@widget`.

There is **no hardcoded built-in widget list** — the tag is the single source of truth. The `@rovy/ui` package is itself built with plain `rbxtsc` (not the transformer; the injected `import __rovyUi from "@rovy/ui"` would be circular), so built-in widgets keep the runtime `widget(fn, "@rovy/ui/name")` wrapper for registration. The `@widget` tag on them exists purely as the consumer-side detection signal.

Conceptual output:

```ts
const Window = RovyUi.__widget(function Window(props: WindowProps): void {
	const style = RovyUi.getActiveStyle();
	print(style.windowBgColor, props.title);
}, { id: "src/ui/Window@Window", name: "Window" });

RovyUi.__scope("src/ui/Window:0", () => Window({ title: "Inventory" }));
```

Widget discovery follows the Rovy side-effect model: `rovy.loadPaths(...)` requires the module so the injected registration runs. Runtime never uses `debug.info` for identity; compile-time keys are mandatory.

## Validation

The shipped package should keep coverage for:

- fixture proving widget wrapping registration
- fixture proving `Window(args)` lowers with stable callsite identity
- fixture proving widget `style: Style` param is removed from the runtime call signature
- fixture proving lowered widget body reads `RovyUi.getActiveStyle()` first
- fixture proving same-file implementation requirement
- fixture proving `StyleScope({ patch }, fn)` only affects widget calls inside `fn`
- fixture proving nested scopes merge partial patches correctly
- fixture proving leaving scope restores the parent active style automatically
- fixture proving scope discriminator participates in runtime scope identity
- fixture proving overload-based clean signatures
- fixture proving `useState` storage is isolated per widget callsite
- fixture proving `useEffect` dependency cleanup behavior
- negative fixture proving class-based widget authoring is not the intended public model
- fixture proving built-in `@rovy/ui` widget calls lower via the `@widget` tag in the package `.d.ts` (no hardcoded name list)
